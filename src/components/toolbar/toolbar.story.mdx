# Toolbar

A colored message block that can be used to emphasize content or used as a notification banner. Use `Message.Header`
to create a header of the block, and `Message.Body` for the body of the block.

The style of the message change depending if the Header is included or not, Play with the Controls in the canvas to
 see how this component behave with different props

## Components

### `Toolbar.Brand`
The left side, always visible, which usually contains the logo and optionally some links or icons

<ArgsTable of={Toolbar.Brand} />

### `Toolbar.Burger`
The hamburger icon, which toggles the toolbar menu on touch devices

<ArgsTable of={Toolbar.Burger} />

### `Toolbar.Menu`
the right side, **hidden on touch devices**, visible on desktop

<ArgsTable of={Toolbar.Menu} />

### `Toolbar.Container`
This is a container for the `Menu.Item` can be align to the `start` or `end` (left or right) using the **align** prop

<ArgsTable of={Toolbar.Container} />

### `Toolbar.Item`
A single item of the toolbar, ussualy rendered as an `a` or a `div`, can be used as a container for `Menu.Link` and `Menu.Dropdown`

<ArgsTable of={Toolbar.Item} />

### `Toolbar.Link`
The default type of children of `Toolbar.Item`. you should use this component in conjunction with `Toolbar.Dropdown` like this, to correctly use dropdowns on the Toolbar

```jsx
<Toolbar.Item href="#">
  <Toolbar.Link>Item</Toolbar.Link>
  <Toolbar.Dropdown>
    <Toolbar.Item href="#">Subitem 1</Toolbar.Item>
    <Toolbar.Item href="#">Subitem 2</Toolbar.Item>
  </Toolbar.Dropdown>
</Toolbar.Item>
```

<ArgsTable of={Toolbar.Item} />

### `Toolbar.Dropdown`
Container for `Toolbar.Item`'s that will be hidden by default, and will be displayed depending of the props of the parent `Toolbar.Item`

<ArgsTable of={Toolbar.Dropdown} />

### `Toolbar.Divider`
An divider for items inside the `Toolbar.Dropdown`

<ArgsTable of={Toolbar.Divider} />
